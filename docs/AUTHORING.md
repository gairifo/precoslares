# Authoring guide

How to add or update content in `lar-ajuda`.

## Yearly constants ritual

Once per fiscal year (typically January, after the Portaria de atualização do IAS publishes in DR):

1. Copy `src/lib/calculator/constants_2026.json` → `src/lib/calculator/constants_YYYY.json`.
2. Update every numeric field for the new year:
   - `ano_referencia`
   - `ias`, `pensao_social_velhice`, `pensao_minima_regime_geral`, `salario_minimo_nacional`
   - `csi.limite_anual_singular`, `csi.limite_anual_casal`, `csi.limite_mensal_max`
   - `complemento_dependencia.regime_geral.{grau_1, grau_2}` and `regime_agricola_nao_contributivo_psi.{grau_1, grau_2}`
   - `irs_art_84_lares.limite_anual`
   - `cuidador_informal.{valor_base_min, valor_base_max}`
3. Update the import in `src/lib/calculator/constants.ts` to point at the new JSON.
4. Run `npm test` — many calculator tests assert specific values; update the expectations alongside the constants.
5. Update the `c=` year displayed in the result page banner — automatic via `CONSTANTS_VERSION`.
6. Update `docs/PERMALINK.md` field reference if any enum changes.
7. Commit as one PR titled `chore: bump constants to YYYY`.

**Existing permalinks** generated against an older constants year continue to decode and produce a result with current values, with a banner: "Este link foi gerado com valores de YYYY-1; recomenda-se recalcular." Tested by `src/components/Wizard.tsx` `banner.kind === "constants_drift"`.

## Adding a deep-dive `/apoios/<slug>` page

(Phase C — currently blocked on IAS-2026 reconciliation; see [the plan](plans/2026-05-10-001-feat-phase-1-soft-launch-bundle-plan.md) Enhancement Summary §"Outstanding question raised by deepening".)

Once unblocked:

1. Create `src/content.config.ts` defining the `apoios` collection with Zod schema (Zod is shipped with `astro:content` — no separate install).
2. Create `src/content/apoios/<slug>.md` following the 8-section template in **Appendix A** of the plan.
3. Frontmatter requires:
   - `title` (SEO `<title>` text)
   - `h1` (visible heading)
   - `description` (≤ 160 chars meta description)
   - `lastUpdated` (ISO date)
   - `apoio_id` (one of the kebab-case slugs: `csi`, `complemento-dependencia`, `atestado-multiusos`, `irs-art-84`, `irs-art-87`, `erpi-acordo`, `adse-iasfa-sad`, `cuidador-informal`)
   - `sources` (array of `{ label, canonical_url, archive_url? }`)
   - `faqs` (array of `{ q, a }`, min 5)
   - `agent_summary` (≤ 500 chars; LLM-friendly TL;DR)
   - `agent_keywords` (string array, optional)
4. Source URLs must point to one of: `seg-social.pt`, `gov.pt`, `dre.pt`, `portaldasfinancas.gov.pt`, `sns24.gov.pt`, `web.archive.org`. The build-time host allowlist test (`src/lib/permalink/__tests__/`) enforces this.
5. PT-PT vocabulary only (no PT-BR landmines — see Appendix A.4 of the plan for the glossary).
6. Run `npm run build` — Zod will fail loudly on missing/invalid frontmatter.

## Adding a new permalink field

1. Add the field to `src/lib/calculator/types.ts` `ApoiosInput`. Always type as `T | null`.
2. Add to `FIELDS` array in `src/lib/permalink/codec.ts` **at the end** (forward-compat).
3. Add to `FIELD_KINDS` map.
4. Add a validator branch in `src/lib/permalink/validator.ts`.
5. Update the schema-stability snapshot test (it will fail; intentional).
6. Update the field table in `docs/PERMALINK.md`.
7. Update the wizard UI in `src/components/Wizard.tsx` to capture the new field.
8. Bump `PERMALINK_VERSION`? **NO** — adding at the end is forward-compatible. Bump only if renaming or removing.

## Plausible event vocabulary

Centralized at `src/lib/analytics.ts`. Current events:

| Event | When |
|---|---|
| `calculator_completed` | Once per session, on wizard submission. **Never** on permalink hydration. |
| `permalink_copied` | Clipboard write resolves successfully. **Never** on rejection. |
| `permalink_loaded` | Hash decodes successfully on mount. |
| `permalink_invalid` | Hash decode fails (carries `kind` prop: `malformed`, `unknown_version`, or `validation_failed`). |

To add an event: add to `Events` const, then call `track(Events.YourEvent, props?)` from any island. Wrapper handles SSR + script-blocked + plausible-throws cases as noop.

## Releasing a public schema

The build emits `dist/schema/apoios-input.json` (JSON Schema, draft-07) so out-of-process LLM agents can declare a tool that calls `calcular()`. Generated by `scripts/emit-schema.ts` at build time.

Update path: edit `scripts/emit-schema.ts` to mirror any change to `ApoiosInput`. Test that the JSON Schema validates a sample input.
